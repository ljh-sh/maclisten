## Add

- `maclisten file` now exposes recognizer **confidence** (`0`–`1`) in its JSON output.
- **Multiple locales** on `file`: pass a short candidate list via repeatable `--locale`
  (`--locale zh-CN --locale en-US`) or stacking shortcuts (`--cn --en`). `file` runs each
  and returns the **highest-confidence** result with a `candidates` scoreboard — for
  transcribing a recording of unknown language. The caller chooses the candidates;
  `maclisten` never iterates every supported locale.
- `--no-pick` — with 2+ locales, emit one JSON line per locale instead of picking the best
  (for an agent / `jq` to judge).
- `--segments` — include a per-segment `{text, confidence, start, duration}` array.
- `--locale` is now repeatable in the CLI parser (backward compatible; single value still
  works, `mic` / `watch` still resolve a single locale).
- Unit tests for option-parser accumulation.

## Note

- Apple's `Speech` framework is single-locale; confidence is a recognizer heuristic (mean
  of per-segment confidences), not a true language ID. Wrong-locale recognition can still
  hallucinate, so keep candidate locales plausible.
- `mic` and `watch` remain single-locale by design.
